# **⚠️ 프로젝트 구조 및 협업 시 유의사항**
> *   **계층형 아키텍처의 한계:** 초기 개발은 빠르나 프로젝트 규모가 커지면 도메인 단위의 응집도가 떨어질 수 있음을 팀원 모두 인지하고 개발을 진행해야 합니다.
> *   **의존성 충돌:** `pyproject.toml`에 패키지를 추가/수정했다면 반드시 `uv.lock`을 업데이트한 뒤 함께 커밋해야 팀원 간 환경 불일치를 막을 수 있습니다.
> *   **Alembic 마이그레이션 충돌:** 팀원들이 각자 브랜치에서 동시에 DB 모델을 수정하면 main 병합 시 Alembic Head(버전 꼬임) 충돌이 발생하기 쉽습니다. DB 스키마 변경은 가급적 한 명씩 순차적으로 진행하거나 담당자를 지정하세요.


# 1. 각 디렉터리의 역할 및 작성 파일

본 프로젝트는 관심사 분리(Separation of Concerns)를 위해 계층형 아키텍처를 사용합니다.

*   **`app/core/`**: 프로젝트 전반의 핵심 설정 및 인프라 구성 (예: `config.py`, `databases.py`, 보안/예외 처리 로직)
*   **`app/models/`**: 물리적 DB 테이블과 1:1로 매핑되는 ORM 엔티티 (예: SQLAlchemy 모델 클래스)
*   **`app/repositories/`**: DB와 직접 상호작용하는 데이터 접근 계층 (예: ORM을 활용한 구체적인 CRUD 쿼리 로직)
*   **`app/schemas/`**: API 요청/응답 시 데이터 구조를 정의하고 유효성을 검증하는 DTO (예: Pydantic 모델 클래스)
*   **`app/services/`**: 핵심 비즈니스 로직 수행 (예: 데이터 가공, 외부 연동, 복잡한 연산 처리 및 API-Repository 간 결합도 완화)
*   **`app/apis/`**: 클라이언트 요청 접수 및 응답을 반환하는 프레젠테이션 계층 (예: API 엔드포인트 URL, HTTP 메서드 정의)

## 2. 주요 파일의 역할

*   **`app/main.py`**: 애플리케이션 진입점(Entry Point). FastAPI 인스턴스를 생성하고 라우터 등록 및 미들웨어(CORS 등)를 설정합니다.
*   **`app/core/config.py`**: 환경 변수(`.env`) 및 전역 설정(DB 접속 URL, 시크릿 키 등)을 관리합니다.
*   **`pyproject.toml`**: 프로젝트 메타데이터 및 요구되는 패키지 의존성을 명시합니다.
*   **`uv.lock`**: 설치된 패키지들의 정확한 버전을 고정하여 팀원 간 완벽히 동일한 실행 환경을 보장합니다.

## 3. 데이터베이스 연결 설정

*   **`app/core/db/databases.py`**: SQLAlchemy를 활용해 DB 엔진(Engine)과 세션(Session) 생성 로직을 작성합니다.
*   `create_engine`으로 DB와 연결하고, API 요청마다 DB 세션을 열고 닫기 위해 의존성 주입(Dependency Injection)용 `get_db()` 제너레이터 함수를 구현합니다.

## 4. SQLAlchemy 모델 작성 및 Alembic 마이그레이션

1.  **모델 작성 (`SQLAlchemy`)**: `app/models/` 내에 SQLAlchemy `Base`를 상속받는 파이썬 모델 클래스를 정의합니다.
2.  **마이그레이션 스크립트 생성 (`Alembic`)**: 터미널에 `alembic revision --autogenerate -m "커밋 메시지"`를 입력하여 변경사항을 추적하는 파일을 생성합니다.
3.  **DB 스키마 적용**: 터미널에 `alembic upgrade head`를 입력하여 생성된 스크립트를 실제 DB에 반영(업데이트)합니다.

## 5. API 구현 흐름

안정적인 개발과 테스트를 위해 DB 계층부터 역순으로 개발을 진행하는 것을 권장합니다.

1.  **Model (`models/`)**: DB 테이블 스키마 정의
2.  **Repository (`repositories/`)**: DB 데이터 조작(CRUD) 로직 작성
3.  **Schema (`schemas/`)**: 데이터 검증을 위한 Pydantic DTO 작성
4.  **Service (`services/`)**: Repository를 호출하여 핵심 비즈니스 로직 처리
5.  **API/Router (`apis/`)**: 클라이언트 요청 접수, Service 로직 호출 및 최종 응답 반환
