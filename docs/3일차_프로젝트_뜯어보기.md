# AH_health_web_development_assignment 개발 가이드

이 프로젝트는 **FastAPI** 백엔드 애플리케이션을 개발하고 배포하기 위한 보일러플레이트(Template)입니다. 현대적인 Python 패키지 및 가상환경 관리 도구인 `uv`(`pyproject.toml`, `uv.lock`)를 사용하며, 데이터베이스 마이그레이션 도구인 `alembic`과 컨테이너화를 위한 `Docker` 환경을 포함하고 있습니다.

본 가이드는 디렉터리 구조에 대한 이해와 로컬 개발 환경 세팅을 돕기 위해 작성되었습니다.

---

## 📂 디렉터리 구조 및 핵심 역할 분석

### 1. 루트 영역 (Root)
*   **`pyproject.toml` & `uv.lock`**: `uv` 도구를 사용한 패키지 의존성 및 가상환경 관리 파일입니다. 기존 `pip`나 `poetry`에 비해 비약적으로 빠른 속도로 의존성을 설치하고 동기화합니다.
*   **`alembic.ini` & `alembic/`**: SQLAlchemy 모델의 변화를 감지하여 데이터베이스 스키마를 버전별로 관리하고 마이그레이션(생성·수정)하는 도구입니다.
*   **`docker-compose.yml` & `app/Dockerfile`**: 데이터베이스(PostgreSQL, MySQL 등)와 FastAPI 웹 서버를 컨테이너로 묶어 로컬 개발 및 실서버 배포 시 한 번에 빌드하고 실행할 수 있도록 정의한 설정 파일입니다.
*   **`.env` (직접 생성 필요)**: 데이터베이스 접속 주소(DB Connection URL), 시크릿 키 등 보안상 민감하고 환경에 따라 달라지는 변수들을 관리하는 파일입니다. **보안을 위해 Git 저장소에는 공유되지 않으므로 로컬에서 직접 작성해야 합니다.**

### 2. `app/` 핵심 및 DB 영역
*   **`app/main.py`**: FastAPI 애플리케이션의 진입점(Entrypoint)입니다. 앱 초기화, 미들웨어 설정, 라우터 등록 등이 이루어집니다.
*   **`app/core/config.py`**: `.env` 파일의 환경 변수를 읽어와 애플리케이션 전역에서 사용할 수 있도록 클래스 형태로 구조화하는 설정 파일입니다. (주로 Pydantic Settings 라이브러리 사용)
*   **`app/core/db/databases.py`**: 데이터베이스 연결 엔진(Engine)과 세션(Session) 생성, API 요청마다 세션을 주입하기 위한 의존성(Dependency Injection) 함수를 구현합니다.
*   **`app/core/db/models.py` (또는 `app/models/`)**: SQLAlchemy 프레임워크를 사용하여 실제 DB 테이블 구조와 매핑되는 파이썬 클래스들을 정의합니다.

### 3. `app/` 레이어드 아키텍처 (Layered Architecture) 영역
이 프로젝트는 코드의 관심사를 분리하고 유지보수성 및 테스트 독립성을 높이기 위해 **계층형 아키텍처**를 따르고 있습니다.
*   **`apis/` (Controller / Router)**: HTTP 요청을 직접적으로 받아 처리하는 라우터들의 집합입니다. 클라이언트와의 직접적인 소통 창구 역할을 합니다.
*   **`schemas/` (DTO - Data Transfer Object)**: Pydantic 라이브러리를 이용하여 요청(Request) 데이터 유효성 검증 규칙과 응답(Response) 데이터 포맷을 정밀하게 다듬는 데이터 모델 파일들이 위치합니다.
*   **`services/` (Business Logic)**: 프로젝트의 핵심 비즈니스 로직(규칙, 계산, 데이터 가공 등)이 모여있는 레이어입니다.
*   **`repositories/` (Data Access)**: 데이터베이스에 직접 접근하여 조회, 추가, 수정, 삭제(CRUD) 쿼리를 수행하는 파일들이 위치합니다. 서비스 레이어는 직접 DB를 다루지 않고 이 저장소 레이어를 호출하여 데이터를 처리합니다.
*   **`templates/` & `static/` / `media/`**:
    *   `templates/`: HTML 템플릿 파일(예: `index.html`)이 위치합니다.
    *   `static/`: CSS, JS, 이미지 등 정적 자원들을 서빙하기 위한 디렉터리입니다.
    *   `media/`: 사용자가 업로드한 이미지나 파일이 임시/영구 저장되는 물리적 공간입니다.

## 🛠️ 과제 시작을 위한 빠른 개발 세팅 가이드 (Quick Start)

가장 먼저 로컬 가상환경을 구성하고 패키지를 설치한 뒤, 서버가 정상적으로 작동하는지 확인해야 합니다. 아래 순서에 맞게 세팅을 진행해 주세요.

### Step 1: `uv`를 활용한 패키지 설치 및 가상환경 활성화
프로젝트 루트 디렉터리에서 터미널을 열고 아래 명령어들을 실행합니다.

```bash
# 1. uv 패키지 관리 도구 설치 (Mac/Linux 기준)
# * Windows 사용자는 공식 문서(https://docs.astral.sh/uv/getting-started/installation/)를 참고해 주세요.
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. 프로젝트 의존성 설치 및 가상환경 생성 (.venv 디렉터리가 자동 생성됩니다)
uv sync

# 3. 가상환경 활성화
# (Mac / Linux 환경인 경우)
source .venv/bin/activate

# (Windows Command Prompt 환경인 경우)
# .venv\Scripts\activate

# (Windows PowerShell 환경인 경우)
# .venv\Scripts\Activate.ps1
```

### Step 2: `.env` 환경 변수 파일 생성
루트 디렉터리에 `.env` 이름의 파일을 직접 생성한 후 아래와 같이 데이터베이스 세팅 및 환경 변수를 채워 넣습니다. (본인의 로컬 DB 설정에 맞춰 변경해 주어야 합니다.)

```env
PROJECT_NAME="AH Health Assignment"
DATABASE_URL="postgresql+psycopg://postgres:password@localhost:5432/ah_health"
SECRET_KEY="your-super-secret-key-change-me"
```

### Step 3: Alembic을 이용한 데이터베이스 마이그레이션
SQLAlchemy 모델 정보를 실제 데이터베이스 테이블 구조에 동기화하기 위해 마이그레이션 명령어를 실행합니다. (데이터베이스 서버 또는 Docker 컨테이너가 켜져 있어야 합니다.)

```bash
# 최초 마이그레이션 버전 파일 자동 생성 (이미 versions/ 디렉터리에 파일들이 있다면 스킵 가능)
alembic revision --autogenerate -m "Init database schemas"

# 데이터베이스에 마이그레이션 최신화 반영
alembic upgrade head
```

### Step 4: 로컬 uvicorn 웹 서버 구동
FastAPI 애플리케이션을 기동하여 제대로 동작하는지 확인합니다.

```bash
# uvicorn 실행 (수정 시 자동 재기동 되도록 --reload 옵션 적용)
uvicorn app.main:app --reload
```

*   **메인 웹 페이지 접근**: 브라우저에서 `http://127.0.0.1:8000` 접속 (`index.html` 렌더링 확인)
*   **인터랙티브 API 문서**: 브라우저에서 `http://127.0.0.1:8000/docs` 접속 (Swagger UI를 통해 API 테스트 진행)

## 💡 개발 팁 (기능 개발 워크플로우)
이 아키텍처 내에서 새로운 기능(예: 회원가입, 건강 기록 추가 등)을 구현할 때는 아래 순서대로 흐름을 잡으면 얽히지 않고 깔끔하게 개발할 수 있습니다.

1.  **`app/core/db/models.py`**: 필요한 테이블 스키마를 정의하고, `alembic`을 이용해 DB에 반영합니다.
2.  **`app/schemas/`**: 클라이언트가 요청 시 보내야 할 필드 규칙(Pydantic)과 API가 반환해 줄 데이터 형식을 설계합니다.
3.  **`app/repositories/`**: 해당 테이블에 쿼리를 날릴 수 있는 데이터 액세스 메서드를 구현합니다.
4.  **`app/services/`**: 핵심 비즈니스 예외 처리 및 비즈니스 요건이 반영된 핵심 로직을 작성하고, 리포지토리를 조합합니다.
5.  **`app/apis/`**: 라우터 경로를 생성하고, 서비스 객체를 주입받아 요청을 넘겨준 뒤 응답 포맷에 맞춰 리턴해 줍니다.
