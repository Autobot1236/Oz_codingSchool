# 각 디렉터리는 어떤 역할을 수행하며, 작성해야 할 파일의 내용


## 1. 각 디렉터리는 어떤 역할을 수행하며, 작성해야 할 파일의 내용

### app/core/

프로젝트 전체에서 공통으로 사용하는 설정과 기반 코드를 관리하는 디렉터리이다.

주로 작성하는 내용:
- 환경변수 설정
- 데이터베이스 연결 설정
- 공통 설정값
- 보안 관련 설정

### app/models/

데이터베이스 테이블과 연결되는 SQLAlchemy 모델을 작성하는 디렉터리이다.

예를 들어 환자 정보를 저장한다면 `patients` 테이블에 대응하는 `Patient` 모델을 작성한다.

### app/repositories/

데이터베이스에 직접 접근하는 코드를 작성하는 디렉터리이다.

주로 데이터 조회, 생성, 수정, 삭제 같은 CRUD 로직을 담당한다. API나 service에서 직접 DB 쿼리를 작성하지 않고 repository를 통해 접근하면 역할이 분리된다.

### app/schemas/

API 요청과 응답에 사용하는 Pydantic 모델을 작성하는 디렉터리이다.

주로 요청 body 검증, 응답 데이터 형식 정의, Swagger 문서에 표시될 schema 정의를 담당한다.

### app/services/

비즈니스 로직을 작성하는 디렉터리이다.

예를 들어 “환자를 등록한다”, “X-Ray 판독 결과를 저장한다”, “위험 환자를 분류한다” 같은 실제 서비스 흐름을 처리한다.

### app/apis/

클라이언트 요청을 받는 API 라우터를 작성하는 디렉터리이다.

URL endpoint와 HTTP method를 정의하고, 필요한 경우 schema로 요청값을 검증한 뒤 service를 호출한다.


## 2. 주요 각 파일 역할

### app/main.py

FastAPI 애플리케이션의 시작점이다.

주요 역할:
- `FastAPI()` 앱 객체 생성
- API router 등록
- middleware 설정
- static 파일 연결
- 서버 실행 시 가장 먼저 로드되는 앱 구성

### app/core/config.py

프로젝트 설정을 관리하는 파일이다.

주로 `.env` 파일에서 환경변수를 읽어오고, 데이터베이스 URL, 비밀키, 개발/운영 환경 설정 등을 관리한다.

### pyproject.toml

Python 프로젝트의 설정 파일이다.

주요 역할:
- 프로젝트 이름과 버전 관리
- 필요한 패키지 의존성 관리
- formatter, linter, test 도구 설정
- 빌드 시스템 설정

### uv.lock

`uv` 패키지 매니저가 생성하는 lock 파일이다.

설치된 패키지의 정확한 버전을 고정해서 팀원들이 같은 개발 환경을 사용할 수 있게 해준다.



## 3.데이터베이스 연결 설정

### app/core/db/databases.py

SQLAlchemy를 사용해 데이터베이스 연결을 설정하는 파일이다.

주요 역할:
- DB 연결 URL 설정
- SQLAlchemy engine 생성
- session 생성
- Base 모델 선언
- FastAPI dependency로 사용할 `get_db()` 함수 제공

API에서는 `Depends(get_db)`를 사용해서 요청마다 DB 세션을 주입받을 수 있다.


## 4. SQLAlchemy와 Alembic

SQLAlchemy는 Python 코드로 데이터베이스 테이블을 표현하고 조작할 수 있게 해주는 ORM이다.

예를 들어 `Patient` 클래스를 만들면 실제 데이터베이스의 `patients` 테이블과 연결할 수 있다.

Alembic은 SQLAlchemy 모델 변경사항을 데이터베이스에 반영하기 위한 마이그레이션 도구이다.


## 5. API 구현 흐름

FastAPI 프로젝트에서 API는 보통 아래 흐름으로 구현한다.
Client
→ app/apis/
→ app/schemas/
→ app/services/
→ app/repositories/
→ app/models/
→ Database

흐름 설명:

클라이언트가 API 요청을 보낸다.
app/apis/의 router가 요청을 받는다.
요청 body가 있으면 app/schemas/의 Pydantic 모델로 검증한다.
router는 실제 처리 로직을 app/services/에 위임한다.
service는 데이터 조회나 저장을 위해 app/repositories/를 호출한다.
repository는 app/models/의 SQLAlchemy 모델을 사용해 DB와 통신한다.
처리 결과를 응답 schema 형태로 변환해 클라이언트에게 반환한다.
